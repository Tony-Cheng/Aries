Aries

Description

In recent years, toxic comments and messages have become a major problem for online social platforms. Studies have shown that people sending and receiving toxic messages regularly have a notable chance of developing depression and other social and mental disorders. Aries is an onlineweb-based chat application tackling the problem of toxic messages by raising awareness of users' own toxic behaviour. By using neural networks, Aries identifies users' toxic messages and notify them of their toxic behaviour. Moreover, Aries compute a score between 0 and 1 for users to gauge their behaviour. Also, Aries has a recommendation system that suggests friends to users with a similar toxicity score.

Technologies

Classification

The Transformer-XL neural network language model is used to classify toxic comments. The dataset used to train the model comes from the competition Jigsaw Unintended Bias in Toxicity Classification on Kaggle. The fast.ai libray is used to train and infer from the model. A serverless api-endpoint is setup on OpenFaas for neural network inferencing. The training files and the api-endpoint files are in this github repo.

Server

Node.js is used for server-side scripting. Docker is used to containerize the server appplications. Nginx is setup as a load balancer and as a reverse proxy to forward requests and connections to the node server. The databases, nginx, and the node server are all hosted on a private cloud. but the DNS service is hosted on AWS Route 53.

Front-end

Database

Two database technologies, MySQL and MongoDB, are used to provide data storage for the application. MySQL is used to store relational data such as the login information or the messages sent. On the other hand, MongoDB provides more flexibility, so it's used to store chat groups.
